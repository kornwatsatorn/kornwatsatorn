## About

I enjoy solving problems that sit between business and technology.

My work often starts with unclear requirements, missing context, difficult bugs, or challenging technical decisions.

I help teams:

* Clarify requirements
* Design maintainable systems
* Model data effectively
* Investigate root causes
* Improve development workflows
* Make confident technical decisions

While I enjoy architecture and system design, I also work directly with implementation when needed, from backend services to frontend applications.
